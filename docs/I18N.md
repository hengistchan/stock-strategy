# Web 国际化

Strategy Lab Web 当前支持：

- `zh-CN`：简体中文
- `en-US`：English

首次访问时跟随浏览器语言，之后使用 `strategy-lab.locale` 保存用户选择。切换语言会同步更新 `<html lang>`、页面标题、界面文案以及 `Intl` 日期和数字格式，不会改变 API 请求字段或回测数据。

## 代码结构

- `frontend/src/i18n/translations.ts`：中英文资源表和翻译键类型。
- `frontend/src/i18n/core.ts`：插值、语言检测和容错持久化。
- `frontend/src/i18n/I18nContext.ts`：React Context 与 `useI18n()`。
- `frontend/src/i18n/I18nProvider.tsx`：应用级 Provider 和文档语言同步。

新增界面文案时，先在 `zhCN` 增加语义化键，再补齐 `enUS`。TypeScript 会在英文资源缺少键时让构建失败。组件通过 `useI18n()` 获取 `t`、`locale` 和 `setLocale`，不要直接写死用户可见文案。

策略参数的 `label`、`description` 和后端原始错误属于用户或策略作者内容，不由前端翻译，避免改变策略语义。

Web 入口 `index.html` 使用 `Cache-Control: no-cache`，确保版本升级后浏览器会重新检查入口文件；Vite 生成的带内容哈希资源仍由浏览器正常复用。
